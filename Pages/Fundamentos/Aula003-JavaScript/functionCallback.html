O que é um Callback?

Um callback em JavaScript é uma função que é passada como argumento para outra função e é executada depois que uma operação é concluída. Callbacks são essenciais em JavaScript, especialmente quando lidamos com operações assíncronas, como requisições a servidores ou temporizadores.

Por exemplo, se você quiser que algo aconteça somente após uma ação do usuário, como um clique em um botão, você pode usar um callback:

function aoClicar() {
  console.log("O botão foi clicado!");
}

const botao = document.querySelector("button");
botao.addEventListener("click", aoClicar);

Nesse caso, aoClicar é a função callback que será chamada quando o evento de clique acontecer no botão.

Eventos e Seus Manipuladores

Eventos em JavaScript são ações que acontecem no navegador, como um clique de mouse, uma tecla pressionada, ou a rolagem da página. Você pode usar manipuladores de eventos (event handlers) para executar funções (frequentemente callbacks) em resposta a esses eventos.

Aqui está uma breve explicação dos eventos mais comuns mencionados:

click:

Descrição: Esse evento ocorre quando um elemento é clicado com o mouse. É um dos eventos mais comuns, usado em botões, links e outros elementos interativos.

Uso comum: Mostrar uma mensagem, enviar um formulário, abrir um modal, etc.

const botao = document.querySelector("button");
botao.addEventListener("click", function() {
  console.log("Botão clicado!");
});

scroll:

Descrição: Ocorre quando o usuário rola a página ou um elemento com barra de rolagem. É útil para criar efeitos visuais que dependem da posição de rolagem, como carregar mais conteúdo ou animar elementos.

Uso comum: Carregar conteúdo infinito, animar elementos enquanto o usuário rola, ajustar uma barra de progresso.

window.addEventListener("scroll", function() {
  console.log("Página rolada!");
});

resize:

Descrição: Ocorre quando a janela do navegador é redimensionada. Este evento é útil para layouts responsivos, onde você precisa ajustar o design conforme o tamanho da janela muda.

Uso comum: Recalcular layouts, ajustar o tamanho de elementos gráficos, etc.

window.addEventListener("resize", function() {
  console.log("Janela redimensionada!");
});

keydown:

Descrição: Ocorre quando uma tecla é pressionada. É útil para capturar entradas de teclado, como atalhos ou interações em jogos.

Uso comum: Capturar atalhos de teclado, jogos, ou qualquer interação baseada no teclado.

document.addEventListener("keydown", function(event) {
  console.log(`Tecla pressionada: ${event.key}`);
});

keyup:

Descrição: Ocorre quando uma tecla é liberada após ser pressionada. Útil para capturar a ação quando a tecla é solta, como em casos onde você deseja executar algo após o usuário terminar de digitar.

Uso comum: Formulários, capturar entradas em jogos, etc.

document.addEventListener("keyup", function(event) {
  console.log(`Tecla liberada: ${event.key}`);
});

mouseenter:

Descrição: Ocorre quando o ponteiro do mouse entra na área de um elemento. Diferente de mouseover, ele não dispara novamente ao mover o mouse sobre elementos filhos.

Uso comum: Iniciar animações, mostrar tooltips (balões de dica), mudar estilos visualmente quando o mouse passa por cima.

const elemento = document.querySelector(".meuElemento");
elemento.addEventListener("mouseenter", function() {
  console.log("Mouse entrou no elemento!");
});

Como Callbacks e Eventos se Relacionam

Quando você associa um evento a um elemento HTML usando addEventListener, você está basicamente registrando um callback para ser executado quando esse evento específico ocorre.

Por exemplo, se você quer que algo aconteça quando o usuário clicar em um botão, você passaria uma função callback para o evento click:

const botao = document.querySelector("button");

function aoClicar() {
  console.log("O botão foi clicado!");
}

botao.addEventListener("click", aoClicar);

Neste exemplo:

  • aoClicar é o callback, que só será executado quando o evento de click ocorrer no botão.
  • O addEventListener liga o evento click ao elemento botao e define aoClicar como a função que deve ser chamada quando o clique acontecer.

Resumo:

  • Callback: Uma função passada como argumento para outra função, executada após a conclusão de uma operação.
  • Eventos: Ações no navegador (como cliques, teclas pressionadas, rolagem) que podem ser "ouvidos" e manipulados através de callbacks.
  • Relacionamento: Eventos em JavaScript usam callbacks para determinar o que deve acontecer quando o evento é acionado, permitindo a criação de interações dinâmicas na página.
